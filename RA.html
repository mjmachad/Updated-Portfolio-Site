Rogers Accessory Redesign

The Rogers Accessory Redesign proposal was a project that began in May of 2021 and was completed in the late half of January 2022. The objective of this project was to create a platform for accessory vendors that would support the increase of accessory types and in doing so provide visibility to end users when purchasing devices and further accessories. In addition to creating an administrative platform for accessory vendors, the accessory flow for users received a refreshed design to meet with best practices with other ecommerce websites. This was done to allow more options across all channels and to support promotions and accessory bundles.

Objective

Working alongside project managers and stakeholders we designed a platform to support the need to increase accessory types and visibility to users purchasing devices and further accessories. As the only designer assigned to this project my role consisted of conducting product thinking, interaction, visual design and prototyping while working unsupervised to bring the solution to life.

Challenge

The initial challenge early on was understanding how this platform was going to be built and who was going to have access to complete these requests. Based on the business requirements there were three main users that would be given permission to add, remove and reject accessories (Super Admin, Accessory Manager, and Accessory Vendor). In terms of the accessory flow redesign the main challenge was making sure that one design would be able to work across multiple Rogers channels, because each respective channel had different styles in how they present their accessories to the customer. So ensuring that the style guide was consistent for each channel was crucial. To ensure all requirements were captured, I worked alongside the project manager to create the user stories needed to develop the high fidelity prototypes. Being in an agile work environment, low and mid fidelity wireframes were not developed, instead we moved right into the high fidelity wireframe and prototyping to meet deadlines.

Solutions

Adminsitration Portals

The first solution was to design a Rogers Direct web admin tool with an accessory management system. This would grant access and permissions to accessory vendors, Rogers Direct fulfillment staff, Rogers super administrators, and Rogers sales admins. This gave them the ability to create and manage new and existing accessories across all Rogers Direct channels.

Promotion Tool

A promotions tool was also developed to assist Rogers with driving sales and user traffic to the different Rogers Direct channels. The promotion tool consists of a specific precondition feature (i.e. user adds iPhone 12 to their cart) which allows you to define a resulting action (i.e. system will enforce a 50% discount on SKU Iphone12GreyCase). Resulting preconditions and actions can apply to all components of a wireless offer (device, plan, addons, accessories), so they can be mixed to create a specific promotion.

Tag Management

To drive sales volume, a Tag Management system was developed, allowing Rogers administrators the ability to group accessories under a specific tag. Items that are tagged will be present in the order flow with the ability for users to specifically look up items using the tag as a specific reference. There were two use case scenarios that had to be considered, the first being able to search for any accessory you want and apply the tag to it. The other is applying specific data rules to target a larger group of products all at once.

User Facing Designs

Finally to capture all the work created in the web admin tool the customer facing accessory order flow was created to display all the elements depicted in the tools. While viewing the old design I wanted to keep some sense of familiarity so that when customers login and see the new layout they are not overwhelmed by all the changes. Initially the accessories were presented in a list view, but I also created a grid view to give a better depiction of the type of accessories available to them.

Conclusion

This project was my first opportunity to develop a solution that will be utilized by actual users. I am very excited to gain user feedback from their experience with the design. This was an exciting project that helped me not only with working independently but also with learning how to compromise on design solutions based on development constraints. The project also gave me the opportunity to present my solutions to stakeholders and gain experience in developing my designs through constructive feedback. Moving forward I would like to make some future enhancements to the design based on current trends in the ecommerce space, as well as learn about any analytics and feedback from the users.
