Rogers MidMarket Enhancements

The goal of this project was to improve the overall user experience of the Rogers MidMarket buy flow. This was done by examining each step a user would experience when purchasing a device. Followed by identifying parts of the flow where UI improvements could be made and then creating a refreshed design showcasing those proposed changes. Once complete the overall look and feel presents a revitalized experience for the user.

Challenge

The challenge undertaking this project was ensuring all three designers were able to come to an agreement when looking for a solution for each phase of the buy flow. When reviewing the current design of the screens we had to make sure that what we were proposing wasn’t going to require a lot of work from a developers view. So being able to restrain ourselves to just highlighting quick UI changes was the larger task at hand. No business requirements were provided for this project so developing a strong sense of attention to detail was essential to complete this project effectively.

Solutions

Account Information

We began by identifying the first step in the buy flow which was the account information section. Immediately we noticed spacing issues with the promotion offer that was displayed. The navigation arrows were too close to the border of the container. Our design solution was to make the container smaller to allow the carousel arrows to not be flushed with the border. Following that we identified inconsistencies within the account information screen and tidied up the appearance to add a sense of cohesion.

Device Selection

The next step was the device selection phase where we noticed a lack of structure when reviewing the device details. Our proposed design solution here was to move the stock status below the device image and details. This was designed so that when the user selects their specific model they can see if the device is in stock or out of stock based on the details they selected. This change creates a more natural flow of the information, as opposed to the previous design with the stock being at the top.

From there once a device is selected they can choose what payment method they wish to have such as financing payment or paying in full. In the old design the presentation of the offers made it appear as if all offers available would be a “Special Offer”. Our solution to this was to create a tag instead of a banner to help distinguish between what offer has a deal or not. We also rearranged the device details to make it more consistent with the previous device selection screen.

Trade In Device

Proceeding the plan selection the user is now given the option to trade in their device. While an optional step the user is presented with a step by step process with how it works in the beginning. In the old design this takes up a lot of real estate on the screen and doesn’t provide a lot of value to the user. The change here was to minimize the icons and allow the text of each step to provide the value. Next when answering the questions you can see in the old design the radio buttons are a jarring to view with the hyperlink blue colour. We opted to change it to a shade of black while also bolding the answer they select to make it more visible.

In the old design there was some white space below the questionnaire that wasn’t used effectively. So the solution here was to move the estimated cost below the questionnaire to create a better flow of information and by changing the colour of the value to green it removed the need to make the text big and bold. This also opened up space to move the disclaimer message below the device selected.

Plan Selection

Once a device is selected the user is prompted to select their plan. The old design showcased a layout in which the user could navigate between tabs to select plans. We kept this style but instead the new tab option makes it clear where the break is between options. Also we made the non selected options a darker shade of gray to make it more visible, compared to the old design where it blends in with the white background We also noticed that the offers themselves were not centered in the card so by reducing the spacing to make them entered we were able to add another plan which will provide for value to the user

Device Protection

Moving on from the trade in, we now have the device protection phase of the buy flow. Another optional stage we can see in the old design the card view extends the length of the container by quite a bit leaving white space to the right of it, with the “No Device Protection” option being hidden right below. In our new design we stretched out the card view to occupy the width of the container and added bullet points to the details to make it easier to read for the user. Additionally adding a separate card for “No device Selection” makes it feel like this is an actual option a user can select and not a second thought placed in there.

Try it Yourself

Conclusion

This project was very enjoyable to work on. Being able to collaborate with other designers and compromise on solutions was something I don't have the opportunity to do often. Having the capability to do so allowed me as a designer to take more constructive feedback while also providing it and being more assertive in my desing decisions. I also had fun analyzing an existing design to see what can be improved on, while following the constraints set out by the existing infrastructure. Moving forward I want to be able to apply these analytical methodologies into my future work to minimize the need for projects like this.
